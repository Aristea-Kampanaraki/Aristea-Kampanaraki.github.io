Construction BI Dashboard

Primavera P6 + Power BI — Progress & Cost Tracking for a 5-Star Hotel Project

An end-to-end construction performance tracking dashboard for a large-scale 5-star hotel development. Scope: to enhance Earned Value Management by connecting established project management tools and procedures to interactive Power BI reporting.

The report covers three interactive pages. The Executive Summary provides a real-time project health overview — overall schedule and cost performance indicators, a project delay alert with key financial metrics, an S-Curve tracking planned versus actual progress and spend over time, and a cost-schedule performance chart positioning each work package by risk level. The Progress page breaks down performance by building section and work package, identifying the top contributors to budget and flagging packages that are behind schedule but carry low budget visibility — the hidden schedule risk. The Gantt page presents the full activity register of 948 tasks with start and finish dates, percentage completion, and schedule performance status per work package, replicating the familiar Primavera P6 layout within Power BI.

The Challenge

Using P6's Excel export channel as a case study, the challenge was to design an ETL pipeline architecture for project control analytics that is not tied to any specific software. The same pattern can be adapted to any scheduling or project management tool that exports to Excel — whether that is Primavera P6, MS Project, or a custom tracker. No ERP-specific connectors required, making reporting accessible regardless of which recording system is preferred.

Key Features

  • EVM Dashboard: SPI, CPI, Earned Value, Actual Cost, Budget at Completion, and project delay tracked across the full programme timeline — giving project control visibility at a glance.

  • S-Curve — Planned vs Actual: Budget, Earned Value, and Actual Cost visualised as cumulative area series over time, with Planned Progress %, Actual Progress %, and Budget Spent % as line overlays — a complete picture of schedule and cost performance in a single chart.

  • Cost-Schedule Performance by Work Package: A bubble scatter chart plotting SPI against CPI per work package, with bubble size representing budget share. Two colour series — Top 5 High Budget Impact and Top 5 Hidden Schedule Risk — make priorities immediately visible without any further analysis.

  • Progress Tables — Top 10 & Hidden Schedule Risk: The Top 10 table ranks packages by budget contribution and shows their weighted schedule impact. The Outside Top 10 table runs a separate filter to surface underperforming packages below the budget visibility threshold — the ones that would go unnoticed until they become a programme risk.

  • Gantt in Power BI: Full activity register replicating the Primavera P6 layout — WBS hierarchy, SPI colour coding per work package, and phase filter buttons for fast navigation across building sections.

  • Narrative Intelligence: : Instead of interpreting numbers, the report tells executives and project managers directly which work packages require action, what level of urgency applies, and what steps to take — displayed as a plain-language briefing on the canvas.

  • Zero-Touch Refresh: Drop a correctly named P6 export into the folder and the entire dashboard updates — no query edits, no manual steps, no metadata entry. The report is published to Power BI Service, making it accessible and shareable across the organisation from any device.

The Outcome

Where does the project stand today? Are we ahead or behind schedule and budget? Which activities are at risk? These are the questions executives and project managers need answered at every reporting period — and this dashboard answers all three without spreadsheets and without manual updates. Drop a new snapshot into the folder and the entire report updates itself. A fully interactive, shareable report delivering real-time project control visibility to any stakeholder outlining what action is required and why.

Technical Highlights

DAX — Advanced Measure Examples

A selection of the technically advanced DAX measures built for this report, each designed to deliver a specific analytical or communication outcome directly within the dashboard.

Filter Info — A context-aware measure that reads the current slicer state and returns a formatted string showing the selected Project, Report Date, WBS, and Building Section. Displayed permanently at the top of every page, it gives every stakeholder instant confirmation of what they are looking at — eliminating the most common reporting error: presenting KPIs under the wrong filter context without realising it.
Work Package Requiring Action — WBS packages ranked by labour share; a Critical Ratio (CR = SPI × CPI) calculated per package; multiple severity lists built with CONCATENATEX. Nested IF branches return a plain-language briefing — heading, instruction, and package list — directly on the canvas. The project manager reads a recommendation, not a table of numbers.
WBS Outside Top 10 — Uses RANKX on % Budgeted Labour to identify packages ranked outside the top 10 by budget share that are still behind schedule (SPI < 0.95). A standard budget-sorted table would never surface these — they look insignificant by cost but carry real schedule risk. This measure makes the invisible visible.

Visual Design & User Experience

How the report looks, behaves, and communicates — from chart design decisions on the Executive Summary to the navigation architecture that keeps every page responsive and in context.

S-Curve — Planned vs Actual — Visual calculations drive three cumulative area series (Budget, Earned Value, Actual Cost) alongside progress percentage overlays. Actual Cost and Earned Value are time-intelligence measures that cut precisely at the selected Report Date — select any past date and the chart rewinds the full programme history accurately.
Custom Tooltip · Drillthrough · Go to Gantt · Report Date Slicer — The scatter chart tooltip surfaces full package context on hover. Drillthrough connects the performance view to the Gantt activity register in one click. The Go to Gantt bookmark button preserves full slicer context on navigation. The date slicer rewinds the entire dashboard to any historical reporting snapshot.
Bookmark Navigation — Briefing / Critical Work — The Executive Summary right panel toggles instantly between the Briefing view (narrative performance summary) and the Critical Work view (action list by severity) using Power BI bookmarks. No page reload, no loss of slicer context.
Bookmark Architecture — The full bookmark panel shows the navigation structure behind the report. Each named bookmark controls a specific layer state — Executive Summary, Progress, Gantt, Briefing, Critical Work, and Restore views — making the report behave like a multi-screen application within a single Power BI file.

Data Pipeline

A fully automated ETL pipeline extracts schedule and resource assignment data from Primavera P6, transforms it through two parallel staging pipelines in Power Query, and loads a clean star schema into Power BI — ready for earned value management, cost tracking, and schedule performance reporting across multiple projects and multiple project phases.

The full pipeline architecture — sentinel pattern, SourceFile injection, DAX calendar, and all engineering decisions — is documented in the companion project.
