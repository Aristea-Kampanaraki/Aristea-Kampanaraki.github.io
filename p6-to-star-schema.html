From Primavera P6 to Star Schema

Construction BI Dashboard — Progress & Cost Tracking for a 5-Star Hotel Project

📊 Power BI 🔧 Power Query (M) ⭐ Star Schema 📋 Primavera P6 📈 EVM 🏗️ AEC
❮ ❯

A fully automated ETL pipeline that extracts schedule and resource assignment data from Primavera P6, transforms it through two parallel staging pipelines in Power Query, and loads a clean star schema into Power BI — ready for earned value management, cost tracking, and schedule performance reporting across multiple projects and multiple project phases.

The Challenge

Using P6's Excel export channel as a case study, the challenge was to design an ETL pipeline architecture for project control analytics that is not tied to any specific software. The same pattern can be adapted to any scheduling or project management tool that exports to Excel — whether that is Primavera P6, MS Project, or a custom tracker. No ERP-specific connectors required, making reporting accessible regardless of which recording system is preferred.

The pipeline refreshes automatically when a new snapshot is dropped into a folder. The star schema brings together the core reporting requirements of all construction projects — schedule performance, cost variance, earned value, and resource utilisation — into a single Power BI dashboard. Add a project, drop a file, and the dashboard updates itself.

Key Features

  • Zero-Touch Refresh: Adding a new project phase requires only dropping a correctly named file into the folder — no query edits, no manual metadata entry.

  • Parallel Staging Pipelines: Tasks and Resources processed through independent staging queries, each the single source of truth for all downstream tables.

  • Time-Phased Cost Distribution: Monthly EVM actuals unpivoted into one row per Activity × Month, enabling full time-intelligence measures across the project timeline.

  • Dynamic Calendar Dimension: Dim_Calendar is built in DAX as the model's single date spine — one slicer drives every report view consistently across all fact tables.

  • Portable Architecture: Parameter-driven folder paths mean the model relocates between environments without any query rewrites.

  • Full Data Lineage: SourceFile injected before combine ensures every record is traceable back to its originating project phase snapshot.

The Outcome

This project is a proposition for how construction companies of any size — small, medium, or large — can take full advantage of their existing project management tools and benefit from real-time reporting on single or multiple project performance at a glance. No new software, no ERP investment, no IT project. Just the Excel exports they already produce, connected to a self-updating Power BI dashboard that gives project control visibility across the entire portfolio.
